# 主题色规范（2026-02-22）

## 说明

当前 UI 主题采用「暖橙主色 + 冷蓝辅助色 + 深色底板」方案，并已做低饱和调优（更耐看、降低视觉疲劳）。

- 主色：`--primary`（橙色）用于主要按钮、激活态、关键强调。
- 辅助色：`--cool-accent`（蓝色）用于渐变混合与高光层次。
- 中性深色：`--background` / `--card` / `--secondary` 构建整体暗色界面。

> 设计基线来源：`src/renderer/styles/globals.css`

## 设置页高亮规则

文件：`src/renderer/components/SettingsDialog.tsx`

- 主导航激活态：`--primary` + `--cool-accent` 的渐变混合，增强当前 tab 感知。
- 子导航激活态：高对比渐变底色 + 深色文字，保证可读性。
- 内容区当前位置：显示 `当前：<子菜单名>` 标识，避免“迷路”。
- 面板切换：使用 `settings-panel-enter` 渐进动画，降低突兀感。

## 交互色规则

- 主操作按钮默认使用低饱和渐变：`primary(0.94)` → `cool-accent(0.76)`，文字统一 `primary-foreground`。
- 激活态优先使用“半透明渐变 + 中等阴影”，避免高饱和大面积铺色。
- 错误态统一走 `destructive`，不再混用其它高饱和告警色。

## 后续改色建议

如果只想整体换主题，优先修改以下 token，不要直接散改组件类名：

1. `--primary`
2. `--cool-accent`
3. `--background`
4. `--card`
5. `--border`

这样可以保留当前组件层级与状态逻辑，只替换颜色体系。
